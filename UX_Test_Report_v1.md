## 🎭 角色 1 · 李明(28岁,男,网球小白,NTRP 2.5)

> **背景**: 刚学网球2个月,教练说大概NTRP 2.5,自己完全搞不清。想找同水平的人周末练练。

### 🗣 用户原声

- "打开App,登录界面挺酷的,那个网球在飘动,感觉挺专业。微信登录和Apple登录点了都说'即将支持',那我只能用手机号了。"
- "验证码界面还行,6个框挺清楚的。不过这个验证码发到哪个号了?上面写的+86 138****8888——等等,这不是我的号啊,我还没输入手机号呢就跳到验证码页面了?"
- "填资料页面,姓名、性别、年龄段都好选。但是到'技术水平'这里卡住了——让我填个数字?NTRP是什么?0.0是什么意思?我教练说我2.5但我不确定..."
- "哦!下面有个'查看NTRP技术分级标准'的链接,点进去看了一下——1.5-2.0纯初学者,2.5-3.0中级初学者...这个说明很有用!我应该填2.5。"
- "但是这个NTRP输入框也太小了吧,要我手动打数字?万一打错了呢?为什么不做成滑块或者下拉选择?编辑资料页面就有滑块啊,注册页怎么用的文本框..."
- "填完了点'完成设定',直接进主页了。没有任何欢迎语、引导教程、也没告诉我接下来干嘛。一下子看到一堆约球卡片有点懵。"
- "看约球列表,每张卡片信息挺全的——时间、地点、人数、NTRP范围都有。我看到一个'NTRP 2.5-3.5'的单打,感觉适合我。"
- "点'报名'按钮弹出确认框,可以给发起人留言,挺好的。确认后出现成功页面,有'联系发起人'按钮可以直接聊天,这个流程很顺畅!"
- "但是我想看看其他报名的人是什么水平的...要点进详情页才能看到参与者列表。从卡片上直接看不出来已经有谁报名了。"
- "有个约球卡片NTRP要求是4.0-5.0,明显比我高,但报名按钮照样是蓝色可点的。我如果不小心点了会怎样?没有任何'水平不匹配'的警告提示。"

### 🐛 发现的问题

| 编号 | 严重度 | 所在页面 | 问题描述 | 复现路径 | 影响 | 建议 |
|------|--------|----------|----------|----------|------|------|
| #001 | 🔴 P0 | 登录→验证码 | 登录页点"手机号码登录"直接跳到验证码页面,没有输入手机号的步骤。验证码页显示硬编码号码"+86 138****8888"（`LoginView.swift:60` 硬编码传参,`LoginView.swift:160` 直接 `showVerification = true`） | 登录页→点手机号码登录→直接到验证码页 | 用户根本没机会输入自己的手机号,整个登录流程不闭环 | 在登录页和验证码页之间增加手机号输入页面 |
| #002 | 🟡 P1 | 注册页 | NTRP技术水平使用文本输入框（`RegisterView.swift:156-170`,`keyboardType(.decimalPad)`）,而 EditProfileView 使用滑块+GeometryReader（`EditProfileView.swift:345-406`）,两处交互不一致 | 注册页→技术水平字段 | 新手不确定自己的水平,手动输入容易出错;两个页面交互方式不一致 | 统一使用滑块(0.5步进),与EditProfileView一致 |
| #003 | 🟡 P1 | 注册页 | NTRP输入框无实时校验反馈。验证仅在点击提交按钮时触发（`RegisterView.swift:508-521` on-submit检查）,用户输入超出范围的值(如8.0或0.0)时无即时提示 | 注册页→技术水平输入→输入超范围值 | 用户可能反复尝试才能知道有效范围 | 输入时实时显示范围提示(1.0-7.0),或在输入框下方常驻显示有效范围 |
| #004 | 🟡 P1 | 首页 | 水平不匹配的约球没有任何视觉提示或警告。NTRP 2.5的用户可以无障碍报名NTRP 4.0-5.0的约球 | 首页→查看NTRP明显高于自己的约球→点报名 | 可能造成水平严重不匹配的尴尬局面,违背App核心价值 | 在报名确认时显示水平匹配度提示,或对不匹配的约球报名按钮标注"水平差异较大" |
| #005 | 🟡 P1 | 首页 | 新用户注册后直接进入首页,没有任何引导/教程/欢迎弹窗,不清楚App的核心功能和操作方式 | 完成注册→进入首页 | 新手用户可能不知道如何开始使用 | 首次登录增加简单的引导流程或气泡提示 |
| #006 | 🟢 P2 | 首页约球卡片 | 约球卡片无法直接看到已报名者信息,必须点进详情才能看到参与者列表和头像 | 首页→查看约球卡片 | 用户需要额外点击才能判断是否要参加 | 卡片上显示已报名者头像叠加(如微信群头像) |
| #007 | 🟢 P2 | 验证码页 | 验证码倒计时文案使用繁体中文"秒後可重新發送",与产品定位(大陆用户)可能不一致 | 登录→验证码页 | 文字风格一致性 | 统一使用简体中文或繁体中文,保持全App一致 |

### ✨ 亮点

- 登录界面视觉设计出色,暗色主题+浮动网球动画很有品质感
- NTRP分级标准指南(`NTRPGuideView`)内容详实,对新手非常友好,5个等级描述清晰
- 约球卡片信息密度恰到好处——时间、地点、人数、NTRP范围、费用一目了然
- 报名流程很流畅:报名→确认留言→成功→联系发起人→聊天,整个链路闭合
- 注册表单必填/选填分区设计好,降低了注册门槛
- 时间冲突检测(`BookedSlotStore`)在报名时弹出toast警告,避免双重预订

## 🧩 设计层面的模式问题

### 1. 表单校验行为三种模式,无一致标准

| 表单 | 提交按钮 | 校验时机 | 校验范围 |
|------|----------|----------|----------|
| RegisterView | 始终可点(颜色变化提示) | On-submit | 全部必填字段 |
| CreateMatchView | 始终可点(始终绿色) | On-submit | 仅AA制金额 |
| CreateTournamentView | 始终可点(始终绿色) | **无校验** | **无** |
| EditProfileView | 始终可点 | **无校验** | **无** |

**应统一为**: 必填项不满足→按钮置灰+`.disabled(true)` + 点击时显示具体缺少哪项

### 2. "即将推出"占位功能过多(共8处)

| 位置 | 功能 | 代码位置 |
|------|------|----------|
| LoginView | 微信登录 | `LoginView.swift:170` |
| LoginView | Apple登录 | `LoginView.swift:174` |
| MyMatchesView | 编辑约球 | `MyMatchesView.swift:238` |
| MyMatchesView | 关闭报名 | `MyMatchesView.swift:244` |
| ChatDetailView | 查看约球详情(群聊) | `ChatDetailView.swift:176` |
| ChatDetailView | 查看群成员(群聊) | `ChatDetailView.swift:178` |
| SettingsView | 修改密码 | 无实际代码(纯展示行) |
| SettingsView | 关联帐号 | 无实际代码(纯展示行) |

**建议**: 未实现的功能不应有可点击的入口,或在入口处标注"即将推出"而非点了之后才告知。SettingsView的情况更糟——连toast提示都没有

### 3. 导航组件样式三重不一致

| 组件 | 不一致表现 |
|------|-----------|
| 返回按钮大小 | 16pt / 17pt / 18pt 三种 |
| 导航栏标题 | Typography.navTitle(17pt bold) vs inline 16-20pt semibold |
| 导航栏背景 | 有的是绿色(RegisterView、PublicProfileView),有的是白色(SettingsView、CreateMatchView) |

### 4. 注册表单 vs 编辑资料 交互不一致

| 字段 | RegisterView | EditProfileView |
|------|-------------|-----------------|
| NTRP输入 | 文本框(手动输入) | 滑块(拖动选择) |
| 头像 | PhotosPicker | 仅视觉占位(camera图标+文字) |
| 性别 | chip选择 | chip选择(样式略不同) |
| 提交校验 | On-submit校验4项 | 无任何校验 |

### 5. 不可交互元素伪装成可交互

设置页的4个`settingsRow`项带有chevron.right箭头,看起来可以点击跳转,但实际上是纯展示的HStack。这违反了iOS设计的基本准则:箭头暗示有下一级页面。

## 💡 竞品参考建议

### 对比 Meetup
- **缺失**: 活动地图视图(Meetup可在地图上浏览附近活动)
- **缺失**: 活动分类/标签系统
- **缺失**: 活动提醒设置(提前X分钟提醒)
- **可参考**: Meetup的RSVP确认+等待列表机制

### 对比 Keep / 咕咚
- **缺失**: 运动数据追踪(打球时长、频率统计图表)
- **缺失**: 成就/徽章的获得条件和进度展示
- **可参考**: Keep的运动日历视图(一目了然每周打球安排)

### 对比 网球汇
- **优势**: Let'sTennis的NTRP分级更专业、球场数据库更丰富
- **缺失**: 球场实景照片和导航
- **缺失**: 约球评价后的水平自动调整(网球汇有ELO算法)
- **缺失**: 活动费用在线支付

### 对比 小红书/社交App
- **缺失**: 动态/朋友圈功能(分享打球照片/战绩)
- **缺失**: 话题/圈子功能(如"周末双打群""初学者交流")
- **可参考**: 小红书的图文内容格式用于球友分享

### 关键缺失UI模式汇总
1. **地图视图**: 按位置浏览附近约球和球场
2. **搜索功能**: 全局搜索球友/约球/球场
3. **日历视图**: 我的约球日程一览
4. **等待列表**: 满员后自动排队
5. **活动提醒**: 自定义提前通知时间
6. **在线支付**: AA费用在线收款
7. **数据统计**: 运动频率、胜负统计图表
